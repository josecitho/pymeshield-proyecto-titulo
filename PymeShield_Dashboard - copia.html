Recomendaciones de seguridad
4 pendientes
1
Cerrar el puerto SSH (22) o restringir acceso por IP
El puerto 22 expuesto a toda la red es un vector de ataque frecuente. Límite el acceso solo a IPs de confianza o deshabilite SSH si no se usa.
2
Deshabilitar RDP o usar VPN para acceso remoto
El escritorio remoto (puerto 3389) no debe estar expuesto directamente a Internet. Use una VPN como capa adicional de protección.
3
Identificar y registrar el dispositivo desconocido
Hay un dispositivo Android sin nombre en la red (192.168.1.88). Si no pertenece a la empresa, desconéctelo y cambie la contraseña del Wi-Fi.
4
Actualizar el firmware del router
La versión actual del router tiene vulnerabilidades conocidas. Actualice al firmware 2.0.3 para corregir fallos de seguridad reportados.
